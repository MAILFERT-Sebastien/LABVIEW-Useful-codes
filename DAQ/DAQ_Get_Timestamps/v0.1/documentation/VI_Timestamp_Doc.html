VI Timestamp

Horodatage hardware des triggers caméra — Documentation technique

USB-6353 CTR3 PFI7 100 MHz Précision 10 ns Rollover géré LabVIEW 2025

01 Objectif

Ce VI enregistre le timestamp absolu de chaque trigger envoyé à la caméra, avec une précision hardware de 10 ns. Les timestamps sont écrits dans un fichier CSV à chaque acquisition, écrasé au démarrage suivant.

Pourquoi un timestamp hardware ?
Le signal de trigger caméra est généré entièrement par le hardware (CTR1 → DO), sans intervention logicielle entre les pulses. Un timestamp logiciel aurait un jitter de plusieurs millisecondes. CTR3 capture chaque flanc montant avec une précision de 10 ns (horloge interne 100 MHz).

02 Principe

CTR3 est configuré en CI Comptage de fronts sur l'horloge 100 MHz. Le signal P0.4 (rebouclé sur PFI7) est l'horloge d'échantillonnage : à chaque flanc montant, CTR3 capture la valeur courante du compteur en ticks.

T_absolu = T0 + (ticks + offset_rollover) / 100 000 000

T0 est l'heure système au moment du Start. L'offset_rollover compense le débordement du compteur 32 bits (rollover toutes les 42.9 s à 100 MHz).

03 Architecture — State Machine

IDLE→ INIT→ START→ RUNNING⇄ WRITE→ STOP→ IDLE

IDLE

Attente du bouton Start. Event Structure sur la valeur du bouton → lit N (nombre de frames) → état INIT.

INIT

Création tâche CTR3, configuration timing, ouverture CSV, enregistrement T0.

  • DAQmx Create Channel — CI Comptage de fronts, /USB-6353/ctr3, edge Montant
  • Nœud propriété CI.ComptFronts.Term = /USB-6353/100MHzTimebase
  • DAQmx Timing — source /USB-6353/PFI7, Échantillons finis, N samples, rate 10 Hz
  • Get Date/Time In Seconds → SR T0
  • Open/Create/Replace File (mode Replace) → SR file ref
  • Écriture header : frame_index,timestamp_iso,timestamp_unix\n

START

⚠ Ordre critique
DAQmx Start Task CTR3 (ce VI) doit être exécuté avant le démarrage des tâches DO/AO du VI principal. Sinon le premier trigger caméra n'est pas capturé.
  • DAQmx Start Task (CTR3)
  • frame_i → 0
  • → état RUNNING

RUNNING

Lecture non-bloquante d'un sample. Boucle sur lui-même jusqu'à réception ou Stop.

  • DAQmx Read — Compteur DBL, 1 voie, 1 échantillon, timeout 0.01 s
  • Case sur l'erreur : True (timeout) → effacer erreur → RUNNING
  • Case sur l'erreur : False (No Error) → ticks → SR → état WRITE
  • Bouton Stop → forcer état STOP

WRITE

Calcul timestamp, gestion rollover, écriture ligne CSV.

  • Si ticks < ticks_prev → rollover → offset += 4 294 967 296
  • ticks_prev ← ticks (valeur brute, sans offset)
  • dt = (ticks + offset) / 100 000 000.0
  • T_absolu = T0 + dt
  • Format Date/Time String %Y-%m-%dT%H:%M:%S.%3u → timestamp_iso
  • Conversion DBL(T_absolu) → timestamp_unix
  • Write to Text File : "%d,%s,%.3f\n"
  • frame_i + 1 — si frame_i ≥ N → STOP, sinon → RUNNING

STOP

  1. DAQmx Stop Task
  2. DAQmx Clear Task
  3. Close File
  4. Reset SR : ticks_prev, offset → 0
  5. → état IDLE

Conversion Python

import pandas as pd
df = pd.read_csv('Timestamp.csv')
# offset LV 1904 → Unix 1970 = 2 082 844 800 s
df['datetime'] = pd.to_datetime(df['timestamp_unix'] - 2082844800, unit='s')
